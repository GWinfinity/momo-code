## Features

- **25+ LLM Providers** вҖ” Deepseek, Zhipu (GLM), Moonshot (Kimi), Claude, GPT-4, Gemini, Doubao, OpenRouter, Groq, Mistral, and more
- **Custom Provider** вҖ” Plug in any OpenAI-compatible API with `MOMO_CUSTOM_*` env vars
- **Model Tiers** вҖ” Zero-config selection: `ultra` / `standard` / `lite`
- **Experience Fast Loop (`/evolve`)** вҖ” Second-level prompt injection via KEP protocol. Tactics distilled from success are auto-selected via Thompson sampling
- **Self-Evolution Training (`/fine-tune`)** вҖ” Hour-level weight improvement via Monte Carlo Graph Search (MCGS) + LoRA
- **Claude Code Interop** вҖ” Seamless migration, inherits `.claude/` config, MCP servers, prompts
- **Local-first** вҖ” Your code never leaves your machine. Open source, auditable
- **Effect-powered** вҖ” Built with Effect for composable, type-safe code

## Quick Start

### 1. Set up API key

```bash
# Generic key (works with any provider)
export MOMO_API_KEY=your-api-key

# Or provider-specific
export MOMO_ANTHROPIC_API_KEY=sk-ant-...
export MOMO_OPENAI_API_KEY=sk-...
```

### 2. Start coding

```bash
# Interactive mode
momo

# One-shot task
momo "Refactor auth to use Effect"

# Use a model tier
momo --model ultra "Complex architecture review"
momo --model standard "Fix the login bug"
momo --model lite "Quick code review"
```

### 3. First run

On first run, momo creates `~/.momo/`:


## Experience Fast Loop (`/evolve`)

The experience fast loop (KEP вҖ” Knowledge Embedding Protocol) is momo's unique **second-level** learning system. Unlike `/fine-tune` which updates model weights over hours, `/evolve` learns and applies knowledge in **seconds** via prompt injection.

### How it works

1. **Observe** вҖ” Extract signals from sessions (test pass/fail, edit accepted/rejected, user corrections)
2. **Distill** вҖ” Convert successful patterns into compact Tactic cards
3. **Select** вҖ” Rank tactics via **Thompson sampling** (Bayesian explore/exploit)
4. **Inject** вҖ” Insert top-k tactics into the system prompt for the current task
5. **Solidify** вҖ” Apply verdict, update Beta distribution statistics
6. **Promote** вҖ” High-confidence tactics graduate to `/fine-tune` curriculum

### Three KEP Assets

| Asset | Description |
|-------|-------------|
| **Tactic** | Compact strategy card with triggers, steps, checks, guardrails |
| **Case** | Successful task record with injected tactics |
| **Ledger** | Append-only audit log (JSONL) |

### Tactic Statistics (Beta Distribution)

Each tactic tracks a Beta(Оұ, ОІ) distribution:
- Оұ = 1 + wins, ОІ = 1 + losses
- **Thompson sampling** for exploration/exploitation balance
- **UCB1** as alternative selection strategy

### Evolution Modes

| Mode | Behavior |
|------|----------|
| `balanced` (default) | Normal explore/exploit trade-off |
| `explore` | Favor new tactics, wider sampling |
| `harden` | Favor proven tactics, tighter selection |
| `convention-only` | Only convention-type tactics |

### Storage

Learned tactics are stored in `~/.momo/experience/`:
- `tactics.jsonl` вҖ” All tactic records
- `ledger.jsonl` вҖ” Audit log of all operations

### Environment Variables

| Variable | Description | Default |
|----------|-------------|---------|
| `MOMO_XP_MODE` | Evolution mode | `balanced` |
| `MOMO_XP_DIR` | Storage directory | `~/.momo/experience` |

## Self-Evolution Training (`/fine-tune`)

The weight slow loop improves momo's **model weights** via fine-tuning. This runs at hour-level timescales.

### How it works

1. **Signal Mining** вҖ” Extract learning signals from sessions
2. **Curriculum Synthesis** вҖ” Build training data (Gold/Hard-neg/Replay slices)
3. **Monte Carlo Graph Search (MCGS)** вҖ” Explore training pipeline space
4. **LoRA Fine-tuning** вҖ” Train candidate model
5. **Ratchet Gate** вҖ” Ensure monotonic improvement

### Commands

```bash
momo /fine-tune              # Diagnose, show proposal
momo /fine-tune run          # Execute training
momo /fine-tune run --dry-run # Preview
momo /fine-tune status       # Check status
momo /fine-tune promote      # Promote candidate
```

### Configuration

| Variable | Description | Default |
|----------|-------------|---------|
| `MOMO_EVOLVE_ENABLED` | Enable self-evolution | `true` |
| `MOMO_EVOLVE_AUTO` | Auto-trigger training | `false` |
| `MOMO_EVOLVE_BUDGET_USD` | Max training budget | `50` |

## Migrating from Claude Code

Zero-friction migration:

1. **Config inheritance** (default ON) вҖ” `~/.claude/settings.json` auto-merged
2. **MCP servers** (default ON) вҖ” `.claude/mcp/` work out of the box
3. **Prompts** (default ON) вҖ” `.claude/prompts/` available

```bash
# Disable inheritance
export MOMO_CLAUDE_CODE_INHERIT=false
export MOMO_ONLY=1
```
