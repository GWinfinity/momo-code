What is momo Code?

momo Code is an AI-powered coding agent CLI that helps you write, refactor, debug, and understand code. It connects to 19+ LLM providers and features a unique self-evolution system (/fine-tune) that improves the agent based on your workflow feedback.

Key Capabilities

  • Multi-Provider — Use Claude, GPT-4, Gemini, OpenRouter, Groq, Mistral, and more
  • Model Tiers — ultra / standard / lite zero-config selection
  • Claude Code Interop — Inherits .claude/ config, MCP, prompts
  • Self-Evolution — /fine-tune uses Pioneer Agent MCGS to improve over time
  • Local-First — Your code never leaves your machine

/evolve Experience Fast Loop

The /evolve command runs the Knowledge Embedding Protocol (KEP) — a second-level learning system that operates in seconds via prompt injection, complementing the hour-level /fine-tune weight training.

5-Step Loop

  1. Observe — Extract signals (pass/fail, edit accepted/rejected, corrections)
  2. Distill — Convert success patterns into compact Tactic cards
  3. Select — Rank via Thompson sampling (Bayesian explore/exploit)
  4. Inject — Insert top-k tactics into system prompt
  5. Solidify — Apply verdict, update Beta distribution statistics

Three KEP Assets

AssetDescription
TacticCompact strategy card: triggers, steps, checks, guardrails
CaseSuccessful task record with injected tactics
LedgerAppend-only audit log (JSONL)

Commands

$ momo /evolve                    # Run evolution
$ momo /evolve --mode=explore     # Favor exploration
$ momo /evolve --mode=harden      # Favor proven tactics
$ momo /evolve --list             # Show learned tactics
$ momo /evolve --inject           # Inject for current task

/fine-tune Self-Evolution

The /fine-tune command enables momo Code to improve its own model using Monte Carlo Graph Search (MCGS), inspired by Pioneer Agent.

Process

  1. Signal Mining — Extract pass/fail/correction signals from sessions
  2. Curriculum Synthesis — Build gold + hard-negative + replay dataset
  3. MCGS Training — EXPAND/SCORE/FUSE/PRUNE pipeline search
  4. Ratchet Gate — Regression check with epsilon=2
  5. Promotion — Hot-swap to better model

Commands

$ /fine-tune              # Diagnose and propose training
$ /fine-tune run          # Execute training
$ /fine-tune status       # Check progress
$ /fine-tune promote      # Promote candidate model
$ /fine-tune rollback     # Roll back to previous
